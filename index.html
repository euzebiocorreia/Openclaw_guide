O agente de IA pessoal mais popular de 2026 — local, hackável e poderoso.

ws://127.0.0.1:18789 — plano de controle central

⚡ OpenClaw vs Nanobot — principais diferenças

Nanobot Husk
  • Python 3.10+
  • Instalação via pip
  • Config em YAML
  • Telegram nativo
  • Framework simples e leve
  • Prompt system via .md
OpenClaw
  • Node.js 22+
  • Instalação via npm global
  • Config em JSON (~/.openclaw/)
  • 20+ canais (Telegram, WhatsApp…)
  • Gateway sempre rodando (daemon)
  • Wizard openclaw onboard
  • Skills auto-instalaveis (ClawHub)
  • Multi-agente com SOUL.md
  • 300k+ estrelas no GitHub 🚀

Passo 1

Pré-requisitos

Verifique se seu Windows está preparado.

ℹ️
Requisitos mínimos Windows 10 v2004 (build 19041+) ou Windows 11. Processador 64-bit. Virtualização habilitada na BIOS.
📦 O que você vai instalar
  • WSL2 (Windows Subsystem for Linux)
  • Ubuntu 24.04 LTS
  • Node.js 22 LTS (via nvm)
  • OpenClaw (npm global)
  • API Key: Gemini ou Claude
  • Bot Token do Telegram
⏱ Tempo estimado
  • WSL2 + Ubuntu: ~15 min
  • Node.js via nvm: ~5 min
  • Instalar OpenClaw: ~3 min
  • Configurar IA + Telegram: ~10 min
  • Wizard onboard: ~10 min
  • Total: ~45 minutos

🔍 Verificar versão do Windows

PowerShell (Admin)
PS> winver # Deve mostrar Windows 10 v2004+ ou Windows 11 # Verificar se virtualização está ativa PS> systeminfo | findstr /i "virtualiz"
02
Passo 2

Instalar WSL2 + Ubuntu

O subsistema Linux onde o OpenClaw vai rodar.

⚠️
Abra o PowerShell como Administrador! Clique com botão direito no menu Iniciar → Terminal (Admin). Sem permissão de admin, os comandos falham.
1

Instalar WSL2 + Ubuntu com um comando

PowerShell (Admin)
PS> wsl --install -d Ubuntu-24.04 # Download ~1GB. Windows vai pedir reinicialização.
2

Reiniciar o PC e criar usuário Ubuntu

Após reiniciar, o Ubuntu abre sozinho pedindo para criar usuário e senha Linux. Anote bem — você vai precisar para o sudo.

3

Confirmar WSL2

PowerShell
PS> wsl --list --verbose # NAME STATE VERSION # * Ubuntu-24.04 Running 2 ← deve ser 2!
✅
VERSION = 2 é obrigatório. Se aparecer 1, rode: wsl --set-version Ubuntu-24.04 2
03
Passo 3

Instalar Node.js 22+

O OpenClaw roda em Node.js — diferente do Nanobot que usa Python. Usamos o nvm para gerenciar versões.

🦞
Por que Node.js e não Python? OpenClaw é escrito em TypeScript/Node.js. É uma escolha de arquitetura que traz o Gateway WebSocket nativo e performance superior para múltiplos canais simultâneos.
💻
A partir daqui, todos os comandos rodam dentro do terminal Ubuntu. Abra pelo menu Iniciar ou pelo Windows Terminal (aba Ubuntu).
1

Atualizar pacotes Ubuntu

Ubuntu Terminal
$ sudo apt update && sudo apt upgrade -y
2

Instalar o nvm (gerenciador de versões Node)

Ubuntu Terminal
$ curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.40.1/install.sh | bash # Recarregar o terminal para ativar o nvm $ source ~/.bashrc
3

Instalar Node.js 22 LTS

Ubuntu Terminal
$ nvm install 22 $ nvm use 22 $ nvm alias default 22 # Verificar $ node --version # v22.x.x ← precisa ser 22+ $ npm --version # 10.x.x
04
Passo 4

Instalar o OpenClaw

Uma linha de comando instala o CLI global. Simples assim.

1

Instalar globalmente via npm

Ubuntu Terminal
$ npm install -g openclaw@latest # Verificar instalação $ openclaw --version # openclaw x.x.x
2

Estrutura criada automaticamente

O OpenClaw cria sua pasta de configuração em ~/.openclaw/ automaticamente no primeiro uso:

~/.openclaw/ ├── openclaw.json ← config principal (API keys, canais) ├── workspaces/ │ └── default/ │ ├── SOUL.md ← personalidade do agente │ ├── memory.md ← memória persistente │ └── todo.md ← lista de tarefas do agente └── skills/ └── ← skills instaladas via ClawHub
3

Verificar saúde da instalação

Ubuntu Terminal
$ openclaw doctor # Exibe diagnóstico da instalação e possíveis problemas

Passo 5

Configurar o Modelo de IA

Obter e configurar a chave de API — escolha Gemini (gratuito para começar) ou Claude (mais sofisticado).

🔵Google Gemini
  • Tier gratuito generoso
  • Gemini 2.5 Flash: rápido e grátis
  • Contexto de 1M tokens
  • Ótimo para começar
🟠Anthropic Claude
  • Raciocínio avançado
  • Excelente para escrita e análise
  • Menor risco de prompt-injection
  • Recomendado para produção
🔵 Gemini
🟠 Claude
1

Acessar o Google AI Studio

Abra aistudio.google.com e faça login com sua conta Google.

2

Gerar a API Key

No menu lateral → "Get API Key" → "Create API Key". Copie a chave gerada (ela só aparece uma vez).

3

Salvar como variável de ambiente

Ubuntu Terminal
$ echo 'export GEMINI_API_KEY="SUA_CHAVE_AQUI"' >> ~/.bashrc $ source ~/.bashrc $ echo $GEMINI_API_KEY # deve printar a chave
1

Criar conta na Anthropic

Acesse console.anthropic.com, crie sua conta e adicione um método de pagamento.

2

Gerar a API Key

Settings → API Keys → Create Key. A chave começa com sk-ant-.... Copie imediatamente.

3

Salvar como variável de ambiente

Ubuntu Terminal
$ echo 'export ANTHROPIC_API_KEY="sk-ant-SEU_TOKEN"' >> ~/.bashrc $ source ~/.bashrc $ echo $ANTHROPIC_API_KEY

Passo 7 — O mais importante!

Wizard de Onboarding

O OpenClaw tem um wizard interativo que configura tudo — Gateway, workspace, canais e skills.

🦞
Este é o grande diferencial do OpenClaw! O comando openclaw onboard guia você passo a passo por toda a configuração de forma interativa. Não precisa editar JSON na mão.
1

Executar o wizard com instalação do daemon

Ubuntu Terminal
$ openclaw onboard --install-daemon # --install-daemon instala o Gateway como serviço systemd # Ele vai iniciar automaticamente ao ligar o WSL
2

Seguir as etapas do wizard

O wizard vai perguntar interativamente:

Wizard interativo
? Qual provedor de IA você quer usar? ❯ Google Gemini Anthropic Claude OpenAI Ollama (local) ? Cole sua API Key: AIzaSy... ? Qual modelo padrão? ❯ gemini-2.5-flash gemini-2.5-pro ? Configurar Telegram? (Y/n) ? Cole o token do bot: 7234567890:AAF... ? Seu Chat ID (allowFrom): 123456789 ✓ Gateway configurado! ✓ Daemon systemd instalado e ativado! ✓ Canal Telegram conectado!
3

Verificar o config gerado

~/.openclaw/openclaw.json (exemplo)
{ "model": { "provider": "gemini", "name": "gemini-2.5-flash", "apiKey": "${GEMINI_API_KEY}" }, "channels": { "telegram": { "token": "${TELEGRAM_TOKEN}", "allowFrom": ["123456789"] } }, "memory": { "enabled": true }, "heartbeat": { "intervalMinutes": 30 } }
4

Auditoria de segurança

Ubuntu Terminal
# Recomendado antes de expor o Gateway à rede $ openclaw security audit --deep
⚠️
OpenClaw tem poder real! Ele acessa arquivos, navega na web, executa código. Sempre revise skills de terceiros antes de instalar e rode o audit antes de expor o gateway externamente.

Passo 8

Personalizar o Agente

O SOUL.md define a personalidade, contexto e comportamento do seu agente.

1

Editar o SOUL.md

Ubuntu Terminal
$ nano ~/.openclaw/workspaces/default/SOUL.md
SOUL.md — exemplo
# Meu Agente Pessoal Você é meu assistente pessoal de IA, sempre disponível via Telegram. ## Personalidade - Direto e objetivo nas respostas - Proativo: avisa sobre tarefas pendentes - Memória de contexto entre conversas ## Contexto do usuário - Nome: [seu nome] - Idioma: Português brasileiro - Fuso horário: America/Sao_Paulo ## Comportamento - Ao receber uma tarefa, quebre em etapas e execute - Resuma ações realizadas de forma concisa - Use emojis com moderação
2

Criar agentes adicionais (multi-agente)

O OpenClaw suporta múltiplos agentes isolados. Cada um tem seu próprio SOUL.md e contexto:

Ubuntu Terminal
# Criar um novo workspace/agente $ mkdir -p ~/.openclaw/workspaces/jarvis $ nano ~/.openclaw/workspaces/jarvis/SOUL.md # Registrar o agente $ openclaw agents add jarvis
3

Instalar skills via ClawHub

Ubuntu Terminal
# Listar skills disponíveis $ openclaw skills search web # Instalar uma skill $ openclaw skills install web-browser # Listar skills instaladas $ openclaw skills list # Ou deixe o agente instalar sozinho! # No Telegram, diga: "instale uma skill para navegar na web" # O agente busca e instala automaticamente via ClawHub
09
Passo Final

Executar o Gateway!

Ligar o motor. Com o daemon, o agente fica ativo 24/7 automaticamente.

1

Iniciar o Gateway manualmente (primeira vez)

Ubuntu Terminal
$ openclaw start # Saída esperada: ✓ Config carregado de ~/.openclaw/openclaw.json ✓ Modelo: gemini-2.5-flash ✓ Gateway ativo: ws://127.0.0.1:18789 ✓ Telegram conectado: @meu_openclaw_bot ✓ Heartbeat: a cada 30 minutos ✓ Agente pronto! Envie uma mensagem no Telegram.
2

Gerenciar o daemon (se instalado com --install-daemon)

Ubuntu Terminal
# Ver status do daemon $ openclaw status # Parar o daemon $ openclaw stop # Reiniciar $ openclaw restart # Ver logs em tempo real $ openclaw logs --follow # Atualizar para a versão mais recente $ sudo npm i -g openclaw@latest && openclaw restart
3

Testar no Telegram

Comandos úteis no chat do Telegram
# Status do agente /status # Ver tarefas agendadas /tasks # Limpar memória da sessão /clear # Pergunta normal para o agente: Olá! O que você consegue fazer? # Instalar skill automaticamente: Instale uma skill para verificar o clima
🎉
Funcionou! Seu agente OpenClaw está online, respondendo no Telegram e rodando 24/7 como daemon no WSL2.
